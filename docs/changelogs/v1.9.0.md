# VoicePi v1.9.0

## Highlights
- Added a more reliable selection review flow so you can reopen and retry refinement without losing the original source text or prompt intent.
- Improved regenerate behavior in the review panel so prompt changes, no-op LLM retries, and insertion targeting behave more predictably across repeated reruns.
- Hardened built-in prompt contracts so refinement and translation treat dictated text as source material instead of answering it like a chat request.

## Added
- Added selection-driven review and rerun support that can open the review panel directly when VoicePi does not have a recent insertion match to reuse.
- Added stable regenerate source tracking for both recent insertions and direct selected-text review sessions.
- Added test-environment runtime guards so transient VoicePi panels stay hidden during automated UI-adjacent tests.

## Changed
- Changed review panel prompt handling so selecting a different prompt updates the coordinator immediately and keeps the pending prompt state aligned with regenerate retries.
- Changed the review panel prompt row copy and hinting to better communicate when Enter inserts and when a regenerate is in progress.
- Changed built-in refinement and translation prompts so request-like dictated text is refined or translated as source text instead of being answered.

## Fixed
- Fixed regenerate retries that previously treated unchanged LLM output as a hard failure even when keeping the prior rewritten result was the correct outcome.
- Fixed editable selection inspection so VoicePi can recover selected text from the field value when accessibility APIs omit the selected-text attribute.
- Fixed recent insertion rewrite tracking so temporary focus changes do not prematurely cancel the pending auto-open flow, and insertion fallback can recover the correct target app bundle more often.

## Breaking Changes
- None
