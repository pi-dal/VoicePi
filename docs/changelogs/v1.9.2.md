# VoicePi v1.9.2

## Highlights
- Added an optional Thinking setting for LLM models so you can explicitly send `enable_thinking` as On, Off, or leave it unset.
- Improved dictionary learning and suggestion extraction so VoicePi can pick up preferred terms more reliably after insertion.
- Reduced response-pipeline overhead so recording results and overlay updates stay more stable during fast sessions.

## Added
- Added a Thinking control in Settings for LLM configurations, with `Not Set`, `On`, and `Off` options for mixed-thinking models.
- Added release-facing latency reporting and benchmark coverage so performance regressions are easier to catch before shipping.

## Changed
- Changed the dictionary learning pipeline to normalize extracted terms more consistently and reduce extractor cost on repeated runs.
- Changed the response pipeline to do less blocking work on the critical path and to gate unnecessary floating-panel refreshes.

## Fixed
- Fixed floating-panel sizing so the panel does not shrink unexpectedly within the same recording session.
- Fixed LLM request payload handling so an explicit Thinking choice is preserved and sent when configured.

## Breaking Changes
- None
