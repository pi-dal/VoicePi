# VoicePi v1.10.2

## Highlights
- Added guided permission setup for Accessibility and Input Monitoring so VoicePi can walk you through the System Settings authorization flow more directly.
- Improved the permission helper panel so it stays inside the System Settings window and closes automatically after the permission is granted.
- Standardized local builds and release scripts on the Swift 6.3 toolchain managed by mise.

## Added
- Added PermissionFlow-based guidance for Accessibility and Input Monitoring from the Settings permissions page and permission follow-up flows.
- Added a small PermissionFlow bridge that presents the VoicePi app bundle as the app to authorize.
- Added Swift wrapper scripts for `swift` and `swiftc` so local scripts consistently use the configured mise toolchain.

## Changed
- Changed the Accessibility and Input Monitoring settings buttons to use guided actions instead of the older generic settings prompt.
- Changed launch-time permission handling so the remaining prompts wait when VoicePi opens the guided Accessibility flow and the user has not granted access yet.
- Changed benchmark and verification scripts to run reliably through the project Swift wrappers.

## Fixed
- Fixed the PermissionFlow helper panel growing outside the System Settings window during the guided permission flow.
- Fixed the helper panel staying open after the requested permission becomes granted.
- Fixed Apple Translate SwiftUI import handling for the Swift 6.3 toolchain.

## Breaking Changes
- None
