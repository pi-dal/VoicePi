# VoicePi v1.10.1

## Highlights
- Fixed VoicePi so desktop app editors such as Codex and ChatGPT Atlas are less likely to fall back to `No Input Field Detected` when their input areas are exposed through nonstandard Accessibility nodes.
- Hardened paste injection timing again so clipboard-backed insertion stays stable in more web and IME-backed input fields.

## Added
- None

## Changed
- Changed editable-target detection to fall back from the missing system-focused Accessibility element to the best writable candidate found in the frontmost window tree.
- Changed default paste injection timing floors to keep short clipboard, keypress, and input-source-settle waits above the reliability thresholds that recent latency tuning cut too aggressively.

## Fixed
- Fixed Codex-style desktop editor windows that expose writable `AXGroup` chains instead of a traditional focused text field, so VoicePi can still target the active input area.
- Fixed another class of flaky paste deliveries caused by overly aggressive pre-paste timing reductions in the text injection path.

## Breaking Changes
- None
