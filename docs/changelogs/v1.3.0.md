# VoicePi v1.3.0

## Highlights
Major overhaul of the prompt system and first-class in-app update support for direct-download installs. The prompt profile and option-group system has been replaced with a more flexible prompt workspace supporting per-app and per-website bindings.

## Added
- **Prompt workspace system**: replaces the prompt profile and option-group system with a unified workspace supporting freeform prompt bodies with fixed prefix/suffix wrapping
- **Bundled starter prompts**: three factory prompts — Meeting Notes, JSON Output, and Support Reply — duplicated before editing so shipped presets stay intact
- **App and website prompt bindings**: App Bundle IDs and Website Hosts let user prompts match specific apps (e.g., `com.tinyspeck.slackmacgap`) or browser tabs by host (e.g., `mail.google.com`, `*.notion.so`)
- **Resolved Prompt preview**: shows the exact editable middle section sent at runtime with VoicePi's fixed prefix/suffix instructions
- **Refinement prompt templates**: structured templates for common refinement scenarios
- **Mode-cycle shortcut**: press the trigger once to start recording, again to stop and paste, and a third time to cycle output mode (Raw / Refinement / Translate)
- **In-app update support via AppUpdater**: direct-download installs now receive automatic in-app updates; Homebrew-managed installs continue to route through Homebrew
- **HomebrewInstallationDetector**: detects whether VoicePi was installed via Homebrew by checking the `/caskroom/voicepi/` path and `brew list --cask voicepi`
- **AppUpdateExperience presentation layer**: card and panel UI for each update phase (idle, checking, updateAvailable, downloading, installing, upToDate, failed) with distinct copy per delivery path
- **Versioned release archive**: published ZIP renamed from `VoicePi-macOS.zip` to `VoicePi-<version>.zip` for AppUpdater compatibility
- **Startup hotkey bootstrap with retry**: 6 attempts at 500ms intervals for robust shortcut registration on launch
- Comprehensive test coverage for update delivery routing, update experience phase transitions, and shortcut monitor bootstrap scenarios

## Changed
- AppUpdateCopy now renders distinct copy for Homebrew vs in-app installer delivery paths
- ReleaseAssetNaming produces versioned ZIP names (`VoicePi-<version>.zip`) for AppUpdater compatibility
- Cask URL updated from `VoicePi-macOS.zip` to `VoicePi-<version>.zip`

## Fixed
- Normalized single-select option groups to produce one prompt fragment
- Removed duplicate `LLMRefinerConfiguration` struct, aliased to `LLMConfiguration`
- Extracted hardcoded poll loop constants into named static properties

## Breaking Changes
- None
