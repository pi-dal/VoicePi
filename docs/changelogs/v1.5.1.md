# VoicePi v1.5.1

## Highlights
This release gives you tighter control over prompt bindings with a new Strict Mode toggle, fixes prompt editor saves so they persist immediately, and refreshes the project website's main hero section.

## Added
- A Strict Mode toggle for prompt bindings so you can decide whether matching app and website bindings override the current Active Prompt automatically.

## Changed
- When you click `Save Prompt` in the prompt editor sheet, VoicePi now persists the change immediately instead of waiting for the settings page's bottom save action.
- Matching app bundle ID conflicts are now detected before saving a prompt binding, with reassignment handled explicitly when needed.
- The project website now uses a redesigned hero section with a desk-scene visual, floating install dialog, and updated install call-to-action copy.

## Fixed
- Closing Settings after using the prompt editor's `Save Prompt` action no longer drops the just-saved prompt changes.
- Prompt routing is now easier to reason about when you want the manually selected Active Prompt to win over automatic bindings.

## Breaking Changes
- None
