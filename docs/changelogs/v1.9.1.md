# VoicePi v1.9.1

## Highlights
- Added a dedicated external-processor result panel so processor shortcut runs can review the generated result before insertion.
- Added source snapshot capture for the external-processor shortcut so VoicePi can pass the current text selection as grounded reference context without changing the stdin contract.
- Improved floating refinement feedback so source previews stay visible during processor runs and the panel flow remains stable across retries.

## Added
- Added a dedicated external-processor review panel with source and result sections, copy actions, and unchanged-result status handling.
- Added one-shot source snapshot capture for processor shortcut sessions, including prompt-side `Source` contract generation and bounded source truncation.
- Added source preview support in the floating panel for external-processor shortcut runs.
- Added regression coverage for source snapshot capture, prompt composition, floating panel source previews, and the external-processor result panel.

## Changed
- Changed the external-processor prompt contract so dictated text is always treated as rewrite source material and selected reference text is attached as explicit background context when available.
- Changed processor-shortcut handling so it records immediately instead of diverting into the selected-text rewrite path when a selection is already present.
- Changed post-processing success handling so successful processor-shortcut runs open the dedicated result panel instead of delivering text directly.

## Fixed
- Fixed selection matching in review flows so VoicePi can still validate the target text when accessibility APIs omit the selected-range attribute but still expose the selected text.
- Fixed floating refinement dismissal timing so the compact overlay stays visible long enough to avoid abrupt panel flicker during fast processor responses.
- Fixed recent-insertion regeneration coverage around source stability and processor review interactions.

## Breaking Changes
- None
