# VoicePi v1.12.0

## Highlights
- Added a new Provider settings area with dedicated ASR and LLM subtabs, making it easier to review and adjust speech and refinement services from one place.
- Introduced a file-first configuration system backed by TOML, including automatic migration from the legacy stored settings format and live config file watching.
- Improved synthetic paste delivery by using a session event tap with a private-state event source, reducing cases where injected text is dropped or ignored.

## Added
- Added a Provider section in Settings with separate ASR and LLM subtabs for service-specific configuration.
- Added a TOML-based file-first configuration store, schema, path handling, and watcher support.
- Added automatic migration from legacy persisted configuration into the new file-first configuration flow.
- Added site documentation and generated screenshots that explain and showcase the new configuration workflow.

## Changed
- Changed the internal source layout to a clearer App, Core, Adapters, UI, and Support structure so ongoing maintenance and future feature work stay easier to reason about.
- Changed settings presentation details in the About section so long build strings truncate more cleanly.
- Changed site media assets to use optimized WebP screenshots for the landing page gallery.

## Fixed
- Fixed synthetic paste delivery behavior by switching text injection to a more reliable event source and tap strategy.
- Fixed configuration continuity during the transition to file-first settings by migrating legacy values forward instead of requiring manual re-entry.

## Breaking Changes
- None
