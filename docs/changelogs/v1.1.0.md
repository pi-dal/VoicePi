# VoicePi v1.1.0

## Highlights
Introduces a provider-aware post-processing pipeline with translation support and aligns shortcut permission handling with macOS behavior.

## Added
- Provider-aware post-processing pipeline with three modes: Disabled, Refinement, and Translate
- Persisted translation-provider selection and target output language
- Apple Translate integration with compile-time and runtime availability gating
- Configurable LLM translation endpoint (routes through the existing LLM API)
- Improved fallback and validation when Apple Translate is unavailable or LLM configuration is incomplete
- Guided permission handoff flow with custom pre-prompts for Microphone, Speech Recognition, Accessibility, and Input Monitoring
- Text segmentation for long transcription inputs to prevent chunking artifacts
- Leading-edge fade mask for overflowing transcript text in the floating capsule
- Standard shortcut support via Carbon RegisterEventHotKey API, eliminating Input Monitoring requirement for common shortcuts (modifier + single key)
- Input Monitoring authorization follow-up polling for up to 120 seconds when awaiting a grant
- Settings section with mode, translation provider, and target language controls
- Menu-bar language presentation reflecting the active post-processing mode
- Compact long transient status messages for better readability

## Changed
- Shortcut permission messaging now explains the standard vs. advanced shortcut distinction clearly

## Fixed
- Aligned permission messaging with actual macOS behavior

## Breaking Changes
- None
