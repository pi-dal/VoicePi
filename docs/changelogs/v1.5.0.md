# VoicePi v1.5.0

## Highlights
This release makes VoicePi more resilient when the frontmost app cannot accept pasted text, lets refinement prompts follow the current app or website automatically, and adds a structured user dictionary that can learn repeat corrections from your edits.

## Added
- A fallback copy panel that preserves the final transcript when VoicePi cannot detect an editable input target.
- Menu-bar actions to capture the current app or website and bind it directly to a refinement prompt.
- A structured user dictionary with persisted entries, reviewable suggestions, and post-injection learning support.

## Changed
- App and website prompt bindings now automatically override the manually selected refinement prompt when the frontmost target matches a saved binding.
- Approved dictionary terms are reused as refinement hints so recurring names and terminology stay more consistent.

## Fixed
- Voice results are no longer effectively lost when recording finishes on a non-editable target.
- Reduced prompt mismatches caused by forgetting to switch refinement prompts after moving between apps or sites.

## Breaking Changes
- None
